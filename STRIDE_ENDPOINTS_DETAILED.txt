File: /d/apps/dyploma/Stride/ui/src/app/core/interceptors/auth.interceptor.ts

Function: authInterceptor: HttpInterceptorFn

Input:
  - req: HttpRequest<any>
  - next: HttpHandlerFn

Process:
  1. Get token from authService.getToken()
  2. Skip token injection for auth endpoints (login, register, google, forgot-password, reset-password)
  3. If token exists and not auth endpoint: Clone request and add Authorization header
  4. Call next(authReq)
  5. Catch HttpErrorResponse with status 401:
     - Check if already refreshing (prevent duplicate calls)
     - Check if error is from refresh endpoint itself
     - Set isRefreshing = true
     - Call authService.refreshToken()
     - On success: Retry request with new token
     - On failure: Logout user

Output: Observable<HttpEvent<unknown>>

Error Handling:
  - 401 with token refresh → retry once
  - 401 on refresh endpoint → logout
  - Other errors → pass through

File: /d/apps/dyploma/Stride/ui/src/app/core/interceptors/logging.interceptor.ts

Function: loggingInterceptor: HttpInterceptorFn

Input:
  - req: HttpRequest<any>
  - next: HttpHandlerFn

Process:
  1. Start performance timer
  2. Generate request ID (random UUID)
  3. Check for existing X-Correlation-ID header
  4. If missing: Clone request, add X-Correlation-ID header
  5. Log request: method, URL, duration, body type
  6. Call next(loggedReq)
  7. On HttpResponse event:
     - Calculate duration
     - Capture X-Correlation-ID from response
     - Log success: status code, duration, response type
  8. On HttpErrorResponse:
     - Calculate duration
     - Capture X-Correlation-ID from error
     - Log error: status code, error message, duration
     - Create HttpError object with stack trace

Output: Observable<HttpEvent<any>>

Data Sanitization:
  - Headers: Redacts Authorization, Password, Token
  - Request logging: Logs method, URL, headers (sanitized)
  - Error logging: Full context including correlation ID
