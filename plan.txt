Plan: Stride — Gamified Educational Platform with AI-Generated Tasks
Date: February 10, 2026

Overview
Stride is a Ukrainian-first, all-ages gamified educational platform that uses a proprietary Adaptive AI Service to generate and dynamically adjust task difficulty based on each student's performance, streaks, and learning state. Built on .NET 10 / Angular 20 / PostgreSQL with free/open-source tooling. The platform fills the #1 gap in the Ukrainian EdTech market — no existing Ukrainian platform combines gamification with AI-powered adaptive learning.

2. Feature Lists

MUST HAVE (MVP)

A) Adaptive AI Service (Custom, Proprietary) — Core Differentiator
- Custom .NET 10 service for task generation and difficulty calibration
- Student performance model tracks: accuracy rate, response time, streak length, topic mastery, session frequency, historical difficulty curve
- Difficulty engine calculates a real-time Difficulty Level (1-100) per student per topic using:
  - Rolling accuracy over last N tasks
  - Current streak multiplier (winning streak increases difficulty, losing streak decreases)
  - Time-decay factor (long absence slightly regresses difficulty)
  - Topic-specific mastery vs global mastery weighting
- Task generation pipeline:
  1) AI generates task templates per topic/difficulty band via LLM
  2) Parameterized templates filled with dynamic values
  3) Human review queue for new template types
  4) Cached task pools per difficulty band for instant delivery
- Adaptive flow:
  - Student answers -> evaluate -> update student model -> select next task at target difficulty
- Fallback: pre-authored task banks where AI quality is insufficient

B) Gamification Core
- Daily streaks with streak freeze and repair
- XP per task (scaled by difficulty)
- Levels and skill mastery tree
- Achievement badges
- Weekly leaderboards (Bronze -> Silver -> Gold -> Platinum -> Diamond)

4. Adaptive AI Service — Technical Design

Flow
- Student completes task
- Evaluate answer (correctness, time, hints)
- Update student performance model
- Calculate target difficulty level
- Select or generate next task at target difficulty
- Serve to student

Difficulty Algorithm Principles
- Winning streak (3+ correct) -> difficulty increases 5-10 per answer
- Losing streak (2+ wrong) -> difficulty decreases 10-15 per answer
- Long absence (>7 days) -> difficulty regresses 10-20% toward baseline
- Flow zone targeting: aim for 70-80% success rate
- Per-topic independence: student can be high in one topic and low in another

6. Decisions (Confirmed)
- Name: Stride
- Target: all ages
- Stack: .NET 10 + Angular 20 + PostgreSQL
- AI approach: self-hosted open-source LLM, parameterized templates, human review
- Architecture: modular monolith initially, split later if needed
- MVP subjects: Mathematics, Ukrainian Language, History of Ukraine, English
- Core differentiator: custom Adaptive AI Service that adjusts difficulty per student
