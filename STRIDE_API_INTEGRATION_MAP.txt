auth.interceptor.ts Logic
Location: /d/apps/dyploma/Stride/ui/src/app/core/interceptors/auth.interceptor.ts

1. Adds Authorization header to non-auth requests
   - Pattern: Authorization: Bearer {token}
   - Excluded endpoints: /auth/login, /auth/register, /auth/google, /auth/forgot-password, /auth/reset-password

2. Handles 401 Unauthorized responses
   - Does not retry if response is from /auth/refresh itself
   - Does not retry if already refreshing (prevents duplicate refresh calls)
   - Calls authService.refreshToken() to get new token
   - Retries original request with new token
   - If refresh fails: Logs out user

3. Token Refresh Flow
   - GET current token from authService.getToken()
   - If 401 and not already refreshing: switchMap to refreshToken()
   - New token received: Retry request with Bearer {newToken}
   - If refresh fails: Logout and throw error

---

logging.interceptor.ts Logic
Location: /d/apps/dyploma/Stride/ui/src/app/core/interceptors/logging.interceptor.ts

1. Tracks request/response timing
   - Captures start time, calculates duration
   - Logs via LoggingService.debug() and .error()

2. Propagates X-Correlation-ID
   - Checks response header for existing correlation ID
   - If missing: Creates new ID and adds to request
   - Updates LoggingService with correlation ID

3. Sanitizes sensitive data
   - Redacts Authorization headers
   - Redacts password fields
   - Function: sanitizeHeaders()

4. Logs error context
   - HTTP method, URL, status code, duration
   - Request/response sizes
   - Correlation ID for tracing
   - Function: createHttpError()
